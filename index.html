// case study

Modularizing a Monolith

Problem

Build times hit 12 minutes. Crashes spiked to 0.8%. Team velocity dropped as feature work got tangled in legacy code. Shipping every 6 weeks.

Constraints

No full rewrite. Live product with 1.2M users. Team of 5. Zero downtime requirement.

Decision

Incremental modularization. Split the monolith into 7 feature modules (auth, payments, notifications, etc) while keeping a shared core. Parallel build. Publish modules as local libraries.

  • Isolated modules by domain, clear APIs between them
  • Gradle build cache + parallel compilation
  • Module-level unit tests run independently
  • 3-month phased rollout: auth → payments → notifications → features
